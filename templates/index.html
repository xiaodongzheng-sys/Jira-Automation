First-Time Flow

The first run is simple: connect, confirm, then preview before you create.

Step 1

Connect Google Sheets

Grant access so the workspace can read your source sheet and write result links back after a successful run.

Expected status: Google Sheets = Connected

{% if google_connected %}
{% else %} Connect Google {% endif %}

Step 2

Configure BPMIS Token

Save your BPMIS API token in Configuration. The workspace uses your saved token first and only falls back to .env when it is blank.

Expected status: BPMIS API = pass in Self-Check

Step 3

Sync, Preview, Then Run

Save your setup in Configuration, sync BPMIS projects where PM PIC or All Involved PM matches your email, then preview and run the BPMIS-to-Jira flow.

Safe habit: Sync first, Preview second, Run last

What This Workspace Reads

Your configured source sheet, one input tab, and the mapped columns that drive BPMIS lookup and run preparation.

Spreadsheet source The Google Sheet and tab defined in your saved configuration.
Eligibility logic Rows are considered eligible when the configured Jira Ticket Link column is still blank.
Mapping model Market, system, title, supporting links, and the mapping rules used to prepare the final payload.

What This Workspace Produces

A full Jira browse URL written back to the source sheet after BPMIS accepts the creation request.

Controlled execution Creates Jira tickets through BPMIS for each eligible row that passes preview and run checks.
Write-back Stores the final browse URL in your configured Jira Ticket Link column.
User-scoped settings Each teammate’s spreadsheet and mapping configuration stays under their own local profile.

Preview or Run

Start with Preview. Only run after the candidate list and resolved mappings look correct.

Sync adds missing BPMIS projects without touching existing rows. Preview never creates Jira. Run only touches rows whose Jira link cell is blank. Project column shows the resolved BPMIS summary used for creation.

Portal does not open {% if shared_portal_enabled %} Ask the portal owner to confirm the host Mac, shared portal process, and Cloudflare Tunnel are all online. {% else %} Run ./scripts/run_team_stack.sh restart, then reopen http://127.0.0.1:5000. {% endif %}
