What This Tool Reads

Your configured spreadsheet, one input tab, and the mapped columns that drive BPMIS issue lookup and Jira creation.

Spreadsheet source The Google Sheet and tab defined in your saved configuration.
Eligibility logic Rows are considered eligible when the configured Jira link column is still blank.
Mapping model Market, summary, optional PRD link, market-specific options, and fixed Jira values.

What This Tool Produces

A full Jira browse URL written back to the spreadsheet after BPMIS accepts the ticket creation request.

Jira creation Creates a Jira ticket through BPMIS for each eligible row.
Write-back Stores the final URL in your configured Jira Ticket Link header.
User-scoped settings Each teammate’s spreadsheet and mapping configuration stays under their own local profile.

Simple Start On Mac

For teammates, the most reliable startup path is still Terminal.

Recommended command ./scripts/run_team_stack.sh start
What it does Starts the local helper, starts the local portal, and prepares the dedicated BPMIS Chrome session.
Daily routine Start the stack, open the portal, preview your rows, then run ticket creation.

Quick Troubleshooting

Portal does not open Run ./scripts/run_team_stack.sh restart, then reopen http://127.0.0.1:5000.
Google is not connected Use the Connect Google button again and make sure your account can open the target spreadsheet.
BPMIS requests fail Log in to BPMIS in the Chrome window opened for the tool, then retry Preview or Run.
