Guided Internal Tool

Jira Creation Automation Tool

This tool helps you turn rows in Google Sheets into Jira tickets through BPMIS. If this is your first time, just follow the steps from top to bottom: connect Google Sheets, confirm your local helper is online, save your mapping, then preview before running.

What this tool reads Your configured Spreadsheet, one Input tab, and the columns you map below.
What this tool creates A Jira ticket in BPMIS for each eligible row whose Jira link cell is still blank.
What this tool writes back The full Jira ticket URL into your configured Jira Ticket Link column.

First Time Here? Follow These 4 Steps

If you are using this tool for the first time, do these four things in order.

You only need to set most of this up once. After that, you will usually just update your Spreadsheet and run Preview or Run.

Step 1

Connect Google

Sign in so the portal can read your Spreadsheet and write Jira links back after creation.

You should see: Google Sheets = Connected

Step 2

Start Local Helper

Keep your own BPMIS login on your own machine. The helper bridges your local BPMIS session to the local portal running on your Mac.

You should see: Local Helper = Connected

Step 3

Save Your Mapping

Tell the tool which Spreadsheet and Input tab to read, and how Jira fields should be filled.

You should do this whenever your Spreadsheet columns or mapping rules change.

Step 4

Preview Before Run

Preview shows which rows will be processed and what summary will be used. Run creates real Jira tickets.

Safe habit: Preview first, Run second

Simple Start On Mac

For teammates, the most reliable way to start the tool is still through Terminal.

Recommended

Open Terminal in the project folder and run ./scripts/run_team_stack.sh start. This starts the helper and the local portal in one step.

Why Terminal is better

Terminal avoids macOS permission and quarantine prompts that sometimes block .command files on first use.

2. Local BPMIS Helper

This helper talks to your own locally logged-in BPMIS session.

Why this is needed: the tool should not directly use a shared BPMIS login. Instead, each person keeps their own BPMIS session on their own machine, and the helper bridges that safely to their local portal.

What “Connected” means: your helper is reachable at the URL below and is ready to receive Jira creation requests.

If Local Helper shows Offline, start your local helper first. If BPMIS requests still fail, log in to BPMIS again in Chrome.

Quick Troubleshooting

Use these quick checks before asking the maintainer for help.

Portal does not open

Check whether your local portal is running on your own Mac and whether you can still open http://127.0.0.1:5000.

Google is not connected

Use the Connect Google button again. If your account is rejected, the maintainer may need to add your email to the team whitelist.

Local Helper is Offline

Start your helper on your own Mac, then confirm http://127.0.0.1:8787/health returns a healthy response.

BPMIS still fails

Open BPMIS in Chrome and confirm you are still logged in before running Preview or Run again.
